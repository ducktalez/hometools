# hometools — Architektur-Überlegung: Zwei Video-Server für iOS

## Das Problem: iOS PWA-Konfiguration

Auf iOS gibt es einen **Trade-off bei PWA-Konfiguration** (nicht unsolvbar, aber noch nicht optimal gelöst):

| Anforderung | `display: standalone` | `display: minimal-ui` |
|---|---|---|
| Installierbar als Home-Screen-App | ✅ | ⚠️ (Web-Link, keine PWA) |
| Fullscreen-Button funktioniert | ✅ | ✅ |
| Background-Audio funktioniert | ❌ | ✅ |
| **PiP (Miniplayer) funktioniert** | ❌ | ✅ ← **die kritische Eigenschaft** |
| Browser-Chrome (minimal) | ❌ | ✅ |

**Die Erkenntnisse:**
- **Was PiP zum Funktionieren brachte:** `display: "minimal-ui"` im Manifest
- **Das Problem damit:** iOS installiert das nicht als echte PWA (nur Web-Link)
- **Kurz:** Benutzer müssen aktuell wählen zwischen:
  - **A) Schöne, installierte App** — aber Media wird beim Minimieren pausiert
  - **B) Alle Media-APIs funktionieren** — aber kein Install als echte PWA (nur Link)

> **Wichtig:** Es ist wahrscheinlich möglich, beides zu haben, wir haben es nur noch nicht geschafft. 
> Die Lösung könnte z.B. in anderen HTML/Meta-Tag-Kombinationen liegen oder einem iOS-spezifischen Flag.

## Lösung (aktuell): Zwei Server-Instanzen

```
Benutzer
   ↓
   ├─→ localhost:8001 (display: standalone)
   │   └─ Schöne App ✨
   │
   └─→ localhost:8002 (display: minimal-ui)
       └─ Vollständiger Funktionsumfang 🔊
```

### Installation & Nutzung

```bash
# Terminal 1: Schöne App
hometools stream-video --port 8001 \
  --env HOMETOOLS_VIDEO_PWA_DISPLAY=standalone

# Terminal 2: Funktionale App
hometools stream-video --port 8002 \
  --env HOMETOOLS_VIDEO_PWA_DISPLAY=minimal-ui
```

### iOS Nutzung

**Szenario 1: Ich will Fullscreen und eine installierte App**
- Safari: `http://localhost:8001`
- "Add to Home Screen" → echte PWA-App installiert
- Fullscreen-Button funktioniert ✅
- Background-Audio funktioniert nicht ❌

**Szenario 2: Ich will Background-Audio und PiP**
- Safari: `http://localhost:8002`
- "Add to Home Screen" → Web-Link (nicht echte App)
- Home-Taste → Audio läuft weiter ✅
- PiP-Miniplayer funktioniert ✅

---

## Alternative: Einzeln-Nutzung

Die `minimal-ui`-Variante ist der aktuelle Kompromiss. Sie ist gerade 
**gut genug** für Hintergrund-Audio-Streaming und bietet auch PiP/Fullscreen-APIs.

Wenn Benutzer die App wirklich installieren wollen (als echte PWA), müssen sie 
den Trade-off verstehen: Dann funktioniert Background-Playback auf iOS nicht.
