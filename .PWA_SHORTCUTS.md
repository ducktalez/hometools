# PWA Shortcuts — Einzelne Filme/Songs auf Home-Bildschirm

## Idee

Statt nur die App zu speichern ("hometools Video"), könnten Benutzer auch **einzelne Filme oder Songs** auf den Home-Bildschirm speichern:

```
Home-Bildschirm
├─ 🎬 hometools Video (App)
├─ 🎬 Borat (Film-Shortcut)      ← NEUES FEATURE
├─ 🎬 The Matrix (Film-Shortcut) ← NEUES FEATURE
├─ 🎵 hometools Audio (App)
└─ 🎵 Bohemian Rhapsody (Song-Shortcut) ← NEUES FEATURE
```

**Bei Klick auf Shortcut:**
- Öffnet direkt diesen Film/Song
- Springt zu dieser Stelle in der UI
- Mit eigenem Thumbnail/Icon

---

## Technologie: PWA Shortcuts

PWA Shortcuts sind standardisiert im `manifest.json`:

```json
{
  "name": "hometools Video",
  "shortcuts": [
    {
      "name": "Borat",
      "short_name": "Borat",
      "description": "Direkter Link zu Borat",
      "url": "/video?id=borat_relative_path",
      "icons": [
        {
          "src": "/thumb?path=borat_relative_path",
          "sizes": "any"
        }
      ]
    },
    {
      "name": "The Matrix",
      "short_name": "The Matrix",
      "url": "/video?id=matrix_path",
      "icons": [...]
    }
  ]
}
```

## Implementation (3 Teile)

### Part 1: Backend — Deep Linking Support

**`video/server.py`:**

```python
@app.get("/")
def video_home(id: str | None = None) -> HTMLResponse:
    """Home endpoint — optional id param for direct film access."""
    items = build_video_index(resolved_library_dir)
    
    if id:
        # Benutzer hat Shortcut geklickt
        # Übergebe das Film-ID an die UI
        return render_video_index_html(
            items, 
            library_dir=resolved_library_dir,
            auto_play_id=id
        )
    else:
        return render_video_index_html(items, library_dir=resolved_library_dir)
```

### Part 2: Frontend — Auto-Play beim Deep Link

**`server_utils.js`:**

```javascript
// Beim Laden: Prüfe ob `?id=...` in URL
const params = new URLSearchParams(window.location.search);
const autoPlayId = params.get('id');

if (autoPlayId) {
    // Der Film wurde direkt über Shortcut aufgerufen
    // Finde ihn in der Library und starte automatisch
    window.addEventListener('load', function() {
        const film = filteredItems.find(i => i.relative_path === autoPlayId);
        if (film) {
            playTrack(filteredItems.indexOf(film));
        }
    });
}
```

### Part 3: UI — "Zu Home-Bildschirm hinzufügen" Button

**Pro Film/Song:**

```html
<button class="add-shortcut-btn" 
        data-title="Borat"
        data-id="Comedy/Borat.mp4">
  📌 Zu Home hinzufügen
</button>
```

**JavaScript:**

```javascript
document.querySelectorAll('.add-shortcut-btn').forEach(btn => {
    btn.addEventListener('click', async function() {
        const title = this.dataset.title;
        const id = this.dataset.id;
        
        // Dynamisch neuen Shortcut ins Manifest schreiben
        await fetch('/api/add-shortcut', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({
                title: title,
                id: id,
                thumbnail_url: `/thumb?path=${encodeURIComponent(id)}`
            })
        });
        
        // iOS: Share Menu mit direktem Link
        if (navigator.share) {
            navigator.share({
                title: `Speichere ${title}`,
                text: `Öffnet direkt: ${title}`,
                url: `/video?id=${encodeURIComponent(id)}`
            });
        }
        // Fallback: Copy to Clipboard
        else {
            const url = `${window.location.origin}/video?id=${encodeURIComponent(id)}`;
            navigator.clipboard.writeText(url);
            alert(`Link kopiert!\nZum Speichern: Safari → Share → Zu Home-Bildschirm`);
        }
    });
});
```

## API Endpoints (Backend)

### `POST /api/add-shortcut`
```json
{
  "title": "Borat",
  "id": "Comedy/Borat.mp4",
  "thumbnail_url": "/thumb?path=Comedy/Borat.mp4"
}
```

**Was passiert:**
1. Backend speichert Shortcut in temporärer DB/Datei
2. Beim nächsten Manifest-Request wird Shortcut inkludiert
3. Service Worker cached den neuen Manifest
4. iOS/Android zeigen neuen Shortcut an

## iOS Nutzungs-Flow

```
1. Benutzer öffnet Film in App
2. Klickt "📌 Zu Home-Bildschirm"
3. iOS-Share-Sheet erscheint
4. "Zu Home-Bildschirm" → Icon erstellt mit Film-Thumbnail
5. Bei nächstem Klick → öffnet direkt diesen Film
```

## Limitationen & Workarounds

### Limitation 1: iOS PWA Shortcuts
- iOS unterstützt PWA-Shortcuts **nur begrenzt** (iOS 16.4+)
- Funktioniert besser auf Android

**Workaround:** 
- PWA-Shortcut für unterstützte Systeme
- Fallback: QR-Code-Generator mit `/video?id=...` Link
- Benutzer scannen QR-Code → Safari → Zu Home

### Limitation 2: Manifest ist statisch
- PWA-Manifest wird gecacht
- Neue Shortcuts nicht sofort sichtbar

**Lösung:**
- Service Worker kann Manifest auf-der-Fly modifizieren
- Oder: Bei Shortcut-Add → App neustarten

### Limitation 3: Thumbnail-URL muss erreichbar sein
- iOS lädt Thumbnail beim Speichern
- Muss über HTTPS sein (auf iOS)

**Lösung:**
- Für lokales Netzwerk: `http://` OK
- Bei Remote-Deployment: HTTPS + echte Thumbnails

## Warum ist das cool?

✅ **Schneller Zugriff** — Favoriten direkt auf Home-Bildschirm
✅ **Native-App-Feeling** — Wie Spotify/Netflix
✅ **PWA-native Feature** — Keine Native App nötig
✅ **Funktioniert offline** — Shortcut + Service Worker cached Film
✅ **Einzelne Elemente** — Nicht nur ganze Apps
