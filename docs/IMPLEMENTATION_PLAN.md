- **MIN_RATING-Schwellenwert-Fix: >= statt >** (2026-04-10) — Bugfix: Bei `HOMETOOLS_MIN_RATING=3` wurden Songs mit genau 3★ fälschlich ausgeblendet, weil der Filter `r > MIN_RATING_THRESHOLD` (strikt größer) statt `r >= MIN_RATING_THRESHOLD` (größer-gleich) verwendete. Semantik jetzt: `MIN_RATING=3` bedeutet „zeige alles ab 3★", d.h. 1-2★ = versteckt, 3★+ = sichtbar, unbewertet = immer sichtbar. Fix in beiden Zweigen von `applyFilter()` (Normal-Modus und Debug-Filter-Modus). Debug-Begründungstext von `≤` auf `<` korrigiert. 1 Test angepasst (`test_min_rating_filter_logic_in_apply_filter`). `architecture.md` aktualisiert.

Nicht klickbar/spielbar (`pointer-events: none, opacity: 0.35`). Track-Count im Header zeigt z.B. „3 tracks (+ 20 ausgeblendet)". Textsuche filtert weiterhin normal. Neue `get_debug_filter()` in `config.py`, Parameter-Pipeline durch `render_media_page()` → `render_player_js()` → JS `DEBUG_FILTER`. CSS: `.debug-filtered`, `.debug-reason`. Beide Server (Audio + Video). 2) **Rating Refresh Log**: Persistentes JSON-Log (`<cache_dir>/rating_refresh_log.json`) speichert pro Ordner wann die POPM-Ratings zuletzt vom Dateisystem gelesen wurden (Timestamp + Statistiken). Neuer `GET /api/audio/refresh-log` Endpoint. `POST /api/audio/refresh-ratings` Response erweitert um `last_refresh` und `folder`. UI zeigt Refresh-Info im Header-Element `#refresh-info` (z.B. „23 Ratings gelesen, 5 aktualisiert (14:30)"). **Bugfix:** `refreshFolderRatings()` JS ignorierte Responses mit `changed === 0` durch Bedingung `!data.changed` (falsy bei 0) — gefixt zu `!data.ratings`. 7 neue Tests, `architecture.md` aktualisiert.

- **Crossfade (Audio)** (2026-04-09) — Nahtlose Song-Übergänge: Ein zweites `<audio>`-Element (`_xfadeAudio`) wird lazy erstellt und spielt den nächsten Track ein, während der aktuelle Track ausgeblendet wird. Konfigurierbar via `HOMETOOLS_CROSSFADE_DURATION` (Env-Var, Default `0` = aus, Max `12s`). `get_crossfade_duration()` in `config.py`, als `crossfade_duration` Parameter durch `render_media_page()` → `render_player_js()` → `CROSSFADE_DURATION` JS-Variable injiziert. Nur Audio-Server (Video hat kein Crossfade). Trigger in `timeupdate`: verbleibende Zeit ≤ Dauer + Schutz gegen kurze Tracks (< Dauer+5s). 20-Schritt sinusoide Volume-Rampe (`0.5 - 0.5*cos(π*p)`). Abbruch bei Pause, manuellem Track-Wechsel. Queue-kompatibel (`_resolveNextForCrossfade` peeked). Rein client-seitig. 7 neue Tests (Config 2, JS-Injection 4, Audio-Server 1). `.env.example` aktualisiert.

- **POPM↔Sterne: WMP-Standard-Mapping** (2026-04-10) — Bugfix: Lineare Konvertierung (`raw/255*5`) durch Windows-Media-Player-Standard-Step-Mapping ersetzt. Neue Funktionen `popm_raw_to_stars()` und `stars_to_popm_raw()` in `audio/metadata.py`. Kanonische Werte: 0→0, 1→1★, 64→2★, 128→3★, 196→4★, 255→5★. Bereichs-basiertes Lesen: 0=unbewertet, 1–31=1★, 32–95=2★, 96–159=3★, 160–223=4★, 224–255=5★. Behebt: Bewertungen aus hometools wurden von externen Tools (foobar2000, MusicBee, Mp3tag, Windows Explorer) falsch gelesen und umgekehrt. Geänderte Stellen: `catalog.py` (build_audio_index), `server.py` (GET metadata, POST rating). Snapshot-Version in `index_cache.py` auf v2 gebumpt — alte Snapshots mit linear-gemappten Ratings werden verworfen und frisch vom Dateisystem gebaut. 23 neue Tests (TestPopmRawToStars, TestStarsToPopmRaw, TestPopmRoundtrip). `architecture.md` aktualisiert.
