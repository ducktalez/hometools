# Offline-Download Feature — Umsetzungsplan

## Ziel
Benutzer können Songs/Videos vom Handy aus herunterladen und offline abspielen — ohne native App.

## Warum PWA reicht (ohne native App)

| Feature | Möglich? | Browser-API |
|---------|----------|-------------|
| Offline-UI | ✅ | Service Worker + Cache API |
| Media-Download | ✅ | Fetch + Blob + IndexedDB |
| Persistent Storage | ✅ | IndexedDB (bis 50MB+) |
| Download-Manager | ✅ | IndexedDB + Progress Events |
| Offline-Abspielen | ✅ | `<audio>` / `<video>` mit local Blob URLs |

**Die einzige Limitation:** iOS Safari hat konservativere Speicher-Quoten (~50MB für installierte PWAs), aber für einen Flug reicht das.

---

## Architektur

```
hometools streaming server (Port 8001)
│
├─ UI (klassisch)
│  └─ Neue Buttons: "📥 Download", "⚙️ Offline-Speicher"
│
├─ Service Worker
│  └─ Cache HTML/CSS/JS für Offline-UI
│  └─ Interceptiert Media-Requests
│
├─ IndexedDB (Browser)
│  ├─ downloads (id, title, stream_url, local_blob_url, size, progress)
│  ├─ library (song/video metadata, sync'd bei Download)
│  └─ cache_metadata (was wurde gecacht)
│
└─ Offline-Modus
   └─ UI lädt Library aus IndexedDB
   └─ <audio>/<video> spielt local blob URLs statt stream URLs
```

### Phase 1: Service Worker erweitern (Cache API)
- HTML/CSS/JS cachen → Offline-UI funktioniert

### Phase 2: Download-Manager (IndexedDB + Fetch)
- Backend-Endpoint: `/api/download/start` 
- Progress-Events: `fetch(stream_url)` mit `.body.getReader()`
- Blob speichern in IndexedDB
- UI: Download-Liste mit Fortschrittsbalken

### Phase 3: Offline-Abspielen
- `<audio src="blob:http://...">` statt `stream_url`
- "Offline-Modus" Toggle in der UI
- Library nur aus IndexedDB laden (kein Server-Call)

### Phase 4: Speicher-Management
- "Speicher freigeben" Button
- Zeige: X MB von 50 MB verwendet
- Auf iOS ggf. mit `requestPersistentStorage()` um höheres Limit bitten

### 3. Download-Manager (JS)

```javascript
async function downloadMedia(streamUrl, title) {
  const db = await openIndexedDB();
  const downloadId = Date.now();
  
  // Eintrag erstellen
  await db.add('downloads', {
    id: downloadId,
    title: title,
    stream_url: streamUrl,
    size: 0,
    progress: 0,
    status: 'downloading',
    blob_url: null
  });
  
  // Fetch mit Progress
  const response = await fetch(streamUrl);
  const reader = response.body.getReader();
  const chunks = [];
  let received = 0;
  
  while (true) {
    const {done, value} = await reader.read();
    if (done) break;
    chunks.push(value);
    received += value.length;
    
    // Progress updaten
    await db.put('downloads', {
      ...download,
      progress: (received / response.headers.get('content-length')) * 100
    });
  }
  
  // Blob speichern
  const blob = new Blob(chunks);
  const blobUrl = URL.createObjectURL(blob);
  await db.put('downloads', {
    ...download,
    blob_url: blobUrl,
    status: 'ready'
  });
}
```

## iOS-spezifische Punkte

> **Wichtig:** Siehe [`.iOS_PWA_DECISIONS.md`](.iOS_PWA_DECISIONS.md) für alle iOS-Limitations und wann eine Native App nötig wird.

### Speicher-Quoten
- **Tab-Kontext:** ~50 MB
- **PWA (Home-Screen-App):** bis zu 50 MB (je nach iOS-Version), mit `requestPersistentStorage()` ggf. bis 1 GB
- **Entscheidungs-Punkt:** >200 MB offline nötig → Native App erwägen

```javascript
// Persistent Storage anfordern (iOS 15+)
if (navigator.storage && navigator.storage.persist) {
  navigator.storage.persist().then(persistent => {
    console.log(`Persistent storage: ${persistent}`);
  });
}

// Speicher-Quota abfragen
if (navigator.storage && navigator.storage.estimate) {
  navigator.storage.estimate().then(estimate => {
    const percentUsed = (estimate.usage / estimate.quota) * 100;
    console.log(`Storage used: ${percentUsed.toFixed(2)}%`);
  });
}
```

### Blob URLs auf iOS
- ✅ Funktionieren in `<audio src="blob:...">` und `<video src="blob:...">`
- ⚠️ Lebensdauer: gelten nur für die aktuelle Session → muss persistiert werden
